A Tech Stack is a list of the langauges, packages, protocols, and frameworks that are used to build an application. This page describes the technologies that are used in this demo app.

This demo app is a client-server application, which means that multiple users (clients) can interact with a singler server. In this application, the client is a web browser and the server is a web server. This app can also be called a web application because users interact with the application via web pages.

The HTML code for this tab is in the templates/tech-stack.html file.

Client Side Tech Stack

HTML
The demo app uses the most modern version of HTML, HTML5.
JavaScript
We use some old Android tablets to access the IRS scouting system, so we only use JavaScript features that were available in version 63 (Dec 2017) of Google's Chrome browser. The should work fine in newer browsers such as Microsofts Edge, Apple Safari, or Mozilla Firefox.
Cascading Style Sheets (CSS)
Cascading style sheets control fonts, colors, margins, padding, borders, and countless other visual aspects of a web page. CSS doesn't have versions in the same sense as HTML or JavaScript. If a CSS feature works in the browsers listed in the JavaScript section and it's defined in the Mozilla CSS reference, then it's fair game.
UIKit CSS Framework
The app uses the UIkit CSS Framework. CSS Frameworks provide sophisticated navigation and formatting with much less effort than what would be required if we were to write all CSS from scratch. UIKit is an attractive framework because it is a minimal framework that does not depend on other frameworks, but still has the features needed for the demo.
Socket.io
Socket.io is a JavaScript module that makes it easier to implement web socket connections. Web sockets are an Internet communications protocol that, like HTTP, can be used by web servers and browsers for communication. With HTTP, web servers can only send information to a web browser after receiving an HTTP request. Web socket connections allow browsers to push information to web clients without first receiving a request. Note that we will only be using Socket.io's client features. We'll use the Flask-SocketIO extension for server-side web socket programming.

Server Side Technology Stack

Python
Python is a dynamic interpreted language that is relatively easy to learn and is well suited for data analysis tasks. The app uses Python version 3.12, which is the 2nd-newest version.
Flask Framework
A Python module for creating dynamic web pages. Dynamic pages are created on the fly, when users request them.
Jinja Templates
Flask is designed to work with the Jinja package, which provides tools to assemble web pages in response to user queries. See the Flask Documentation or the Jinja Documentation for more information on Jinja templates.
Flask-SocketIO
Flask-SocketIO is an extension to the Flask framework that enables communication via web sockets. It works with the Socket.io JavaScript framework. See the official Flask-SocketIO documentaion for more information on Flask-SocketIO.
Eventlet Server
Eventlet is a Python networking library that contains web server functionality and supports web socket communications. We don't neet to interact with Eventlet directly, since the Flask-SocketIO package will automatically start an Eventlet web server if it's installed. See the comments in tempates/index.html and the conda-setup.ps1 file for instructions on how to set up you Python environment so that it includes Eventlet.

Scouting System Tech Stack

The scouting system uses all of the components listed above and a few more, including:

  • SQLite
    A simple, effective relational database.
  • SQlite Python Package
    The sqlite package allows us to interact with SQLite databases via Python code.
  • Pandas
    A Python package for manipulating tabular data.
  • Plotly
    A Python package for creating interactive data visualizations.
  • Dash
    A Python package for creating interactive web applications. Dash and Plotly are designed to work together.
